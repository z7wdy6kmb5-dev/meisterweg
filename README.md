# Meisterweg

FCシリーズ（EA Sports FC）監督キャリアの記録管理Webアプリ。フロントエンドのみのSPA。データは端末のブラウザ（IndexedDB）に永続保存します。

## 技術構成
- React + TypeScript + Vite
- IndexedDB（Dexie.js）でデータ永続化
- テーマはCSS変数で管理（ライト／ダーク／チーム12色）
- Vercel（静的ホスティング）への配信を想定

## セットアップ
```bash
npm install
npm run dev      # 開発サーバー
npm run build    # 本番ビルド（tsc 型チェック + vite build）
npm run preview  # ビルド結果のプレビュー
```

## Vercelへのデプロイ
リポジトリを Vercel に接続すれば `git push` で自動デプロイされます。フレームワークは Vite として自動検出されます（ビルド: `npm run build` / 出力: `dist`）。`vercel.json` でSPAのルーティングを設定済み。

## データについて
データはこの端末のブラウザ内（IndexedDB）に保存されます。サーバーには保存されません。
- タブやブラウザを閉じても、PCを再起動しても消えません。
- コードを更新して再デプロイしてもユーザーデータは消えません（コードとデータは独立）。
- 別端末・別ブラウザには引き継がれません。バックアップ／移行は段階7のJSONエクスポート/インポートで行います。

- [x] **段階3**: シーズンスタッツ（集計値・現OVR・OVR増減の自動計算）＋列の表示/非表示
- [x] **段階4**: 移籍（退団モーダル連動・復帰・履歴）
- [x] **段階5**: チーム成績、シーズンメモ
- [x] **段階6**: 縛り（自動判定→手動チェック）
- [x] **段階7**: エクスポート／インポート（Markdown＋JSON）

## ソース構成
```
src/
  types.ts          全エンティティの型定義（段階1で確定）
  db.ts             Dexie DB定義（全テーブルを段階1で定義）
  migrations.ts     スキーマバージョニング／マイグレーション基盤
  repo.ts           データアクセス層（段階1: Career/Season）
  theme.ts          テーマ定義とCSS変数の適用
  AppContext.tsx    選択中キャリア／グローバルシーズンの状態
  App.tsx           ルート（起動ゲート⇄シェル、タブ切替）
  main.tsx          エントリ（テーマ適用→マイグレーション→マウント）
  components/        UI コンポーネント
```
